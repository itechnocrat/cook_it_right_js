Информация становится знанием только тогда, когда выстраивается в структуру/систему.  

Результирующие знания и навыки, как правило, последовательны в человеческой памяти, но их приобретение, как правило, непоследовательный (асинхронный) процесс.

Предлагаю "Структуру и последовательность изучения JavaScript".

При изучении любого языка программирования необходимо придерживаться одного или нескольких принципов изучения:  
- от простого, к сложному,  
- от малого, к большому,  
- от общего, к частному,  
- от частного, к общему.  

Предлагаю, сначала взглянуть на JS, его расширения и инструменты программирования "в общем", и в порядке следования понятий "от простого, к сложному", а затем начать изучать JS по принципу "от частного, к общему", и "от малого, к большому".

"Структура JavaScript и последовательность его изучения" смотри в файле  
`the_structure_of_JavaScript_and_the_sequence_of_its_study.md`  

В "Структуре и последовательности изучения JavaScript" надо усматривать два "измерения":  
- Развитие категорий языка от простых к сложным  
- Порядок изучения категорий языка, расширений языка, инструментов программирования  

Взгляд "в общем" позволит:  
- оценить объем материала,  
- составить график изучения.

Предлагаемые принципы изучения позволят не увязать в материале, т.к. сначала предлагается изучить простые категории, которые легко будет понять благодаря имеющемуся повседневному опыту, а затем, уже, с помощью их, осваивать более сложные категории, которые в повседневной жизни встречаются не часто.  

В JavaScript, как и в любом другом языке программирования, возможно выделить "ядро", которое целесообразно изучить в первую очередь:  

- Состав языка  
    - Алфавит  
    - Лексика  
    - Синтаксис  
    - Семантика  
- Data Types (Типы данных)
    - Primitive Values (Примитивные значения)  
        - Number  
        - Bigint  
        - String  
        - Boolean  
        - Undefined  
- Operators (Операции/Действия с/над типами данных)  
    - Arithmetic (Арифметические)  
        - `+` (addition)  
        - `-` (subtraction)  
        - `/` (division)  
        - `*` (multiplication)  
        - `%` (modulo)  
        - `**` (exponentiation)  
    - Concatenating strings (Конкатенация/Объединение строк)  
        - `+`  
- Variables (Переменные)  
- Declarations (Объявление переменных)  
    - let  
- Инициализация переменных  
    - Assignment operators (Назначение, связывание)  
        - `=`  
    - Лево- и Право- сторонние выражения  
- Литералы  
    - Числовые  
    - Строковые    
- Control Structures (Управляющие структуры)  
    - Sequence (Последовательность)  
    - Selection (Ветвление)  
        - Operators (Операции/Действия с/над типами данных)  
            - Equality (Равенства)  
                - `==`  
                - `===`  
                - `!=`  
                - `!==`  
            - Comparison (Сравнения)  
                - `>`  
                - `<`  
                - `>=`  
                - `<=`  
            - Binary logical operators (Логические)  
                - `&&`  (Logical AND)  
                - `||`  (Logical OR)  
        - if statement  
        - if-else statement  
        - if-else if statement
        - switch statement  
    - Iteration/Loops (Циклы)  
        - while  
        - do-while  
        - for  
    - labeled  
    - break  
    - continue  

Параллельно с "ядром категорий" из JavaScript, необходимо освоить "тонкую оболочку" из расширений языка и инструментов программирования.  
Это самый подходящий момент, т.к. "ядро категорий" из JavaScript не создаст высокой когнитивной нагрузки и позволит начать применять расширения и инструменты языка к уже изученным структурам языка, а это уже очень значительная возможность практики.  

Расширения языка и инструменты программирования:  

- IDE (Интегрированная Среда Разработки)  
    - Обеспечение качества кода  
        - Оформление кода  
            - Formatter (Prettier)  
        - Контроль качества кода
            - Linter (ESLint)  
        - Проверка орфографии  
            - Code Spell Checker  
- Version Control System или Revision Control System (Система контроля версий/Система управления версиями)  
    - GIT - Компас для изучения:  
        - https://ru.wikipedia.org/wiki/Git  
        - https://git-scm.com/  
        - https://githowto.com/ru  
        - https://learngitbranching.js.org/  
        - https://ohmygit.org/  
    - Несколько сложнее, зато подробно и про все сразу:
        - https://docs.gitlab.com/ee/gitlab-basics/  
- GitHub (Хранилище репозиториев)  
    - CLI-утилита для GitHub - [gh](https://cli.github.com/)  
- Debuging (Отладка кода)  
- Комментирование кода  
- Генераторы документации (Программы для создания документации кода)  
- Модульность JS  

После этого станет возможным **КОМФОРТНОЕ** погружение на любую глубину теории JavaScript.  

Итогом изучения "ядра категорий" будет изучение фундаментальных терминологии программирования и понятий JavaScript.  
Это даже не акваланг и не батискаф, это подводная лодка для погружения в JavaScript.

[теорему Бёма — Якопини](https://ru.wikipedia.org/wiki/%D0%A2%D0%B5%D0%BE%D1%80%D0%B5%D0%BC%D0%B0_%D0%91%D1%91%D0%BC%D0%B0_%E2%80%94_%D0%AF%D0%BA%D0%BE%D0%BF%D0%B8%D0%BD%D0%B8#:~:text=%D0%A2%D0%B5%D0%BE%D1%80%D0%B5%D0%BC%D0%B0%20%D0%91%D1%91%D0%BC%D0%B0%20%E2%80%94%20%D0%AF%D0%BA%D0%BE%D0%BF%D0%B8%D0%BD%D0%B8%20%E2%80%94%20%D0%BF%D0%BE%D0%BB%D0%BE%D0%B6%D0%B5%D0%BD%D0%B8%D0%B5%20%D1%81%D1%82%D1%80%D1%83%D0%BA%D1%82%D1%83%D1%80%D0%BD%D0%BE%D0%B3%D0%BE,%D1%81%D1%82%D1%80%D1%83%D0%BA%D1%82%D1%83%D1%80%20%D1%83%D0%BF%D1%80%D0%B0%D0%B2%D0%BB%D0%B5%D0%BD%D0%B8%D1%8F%3A%20%D0%BF%D0%BE%D1%81%D0%BB%D0%B5%D0%B4%D0%BE%D0%B2%D0%B0%D1%82%D0%B5%D0%BB%D1%8C%D0%BD%D0%BE%D0%B9%20(%D0%B0%D0%BD%D0%B3%D0%BB.)).
