Информация становится знанием только тогда, когда выстраивается в структуру/систему.  

Результирующие знания и навыки, как правило, последовательны в человеческой памяти, но их приобретение, как правило, непоследовательный (асинхронный) процесс.

До какого-то момента информация будет дрейфовать одиночными островками по белой карте теории JavaScript, но однажды они соединятся в большой материк, а затем образуют и континент твердой почвы, на которой станет возможна практика.  

При изучении любого языка программирования необходимо придерживаться одного или нескольких принципов изучения:  
- от простого, к сложному,  
- от малого, к большому,  
- от общего, к частному,  
- от частного, к общему.  

Предлагаю, сначала взглянуть на JS, его расширения и инструменты программирования "в общем", и в порядке следования понятий "от простого, к сложному", а затем начать изучать JS по принципу "от частного, к общему", и "от малого, к большому".

В "Структуре и последовательности изучения JavaScript" надо усматривать два измерения:  
Порядок следования понятий - это одно, а порядок их изучения - это другое.  

Взгляд "в общем" позволит:  
- оценить объем материала,  
- составить график изучения.

Предлагаемые принципы изучения позволят:     
- предотвратить увязание в материале и отставание от графика изучения.  

В JavaScript, как и в любом другом языке программирования, возможно выделить "самое малое", которое целесообразно изучить в первую очередь:  
- Состав языка  
    - Алфавит  
    - Лексика  
    - Синтаксис  
    - Семантика  
- Data Types (Типы данных)
    - Primitive Values (Примитивные значения)  
        - Number  
        - Bigint  
        - String  
        - Boolean  
        - Undefined  
        - Null 
- Литералы  
    - Числовые  
    - Строковые  
    - Template literals/interpolated values (Шаблонные литералы)  
        - Нетегированные  
- Операции/действия над значениями типов данных  
    - Арифметические  
        - `+` (addition)
        - `-` (subtraction)
        - `*` (multiplication)
        - `/` (division)
        - `%` (modulo)
    - Comparison Operators (Сравнения)  
        - `==`, `===`, `!=`, `!==`, `>`, `<`, `>=`, `<=`  
    - Logical Operators (Логические операции)  
        - `&&`, `||`, `!`  
    - Conditional (Ternary) Operator (Условная операция)  
        - (condition) ? expressions_1 : expressions_2  
    - Конкатенация  
- Variables  
- Объявление переменных  
    - let  
    - const  
- Лево- и Право- сторонние выражения  
- Инициализация переменных  
- Control Structures (Управляющие структуры)  
    - Sequence (последовательность)  
    - Selection (ветвление)  
        - if statements [ES1]  
        - switch statements [ES3]  
    - Iteration (Циклы)  
        - while loops [ES1]  
        - do-while loops [ES3]  
        - for loops [ES1]  
    - labeled  
    - break  
    - continue  
- Conditional (Ternary) Operator (Условная операция)  

Параллельно с "самым малым" из JavaScript, необходимо освоить "самое малое" из расширений языка и инструментов программирования.  
Это самый подходящий момент, т.к. "самое малое" из JavaScript не создаст высокой когнитивной нагрузки и позволит начать применять расширения языка и овладевать применением инструментов программирования.  
Это возможно сравнить с оснащением подводной лодки:  

- Комментирование кода  
- Debuging - Отладка кода  
- Testing - Tестирование кода  
- Модульность JS  
- Документирование кода (JSDoc)  
- Контроль версий (Git)  
    - https://ru.wikipedia.org/wiki/Git  
    - https://docs.Gitlab.com/ee/Gitlab-basics/start-using-Git.html  
    - https://Git-scm.com/  
    - https://Githowto.com/ru  
- GitHub (Онлайн Хранилище репозиториев)  
    - CLI-утилита для GitHub - [gh](https://cli.github.com/)  

После этого станет возможным **КОМФОРТНОЕ** погружение на любую глубину теории JavaScript.  

Итогом изучения "самого малого" будет изучение фундаментальной терминологии программирования и понятий JavaScript.  
Это даже не акваланг и не батискаф, это подводная лодка для дальнейшего погружения в JavaScript.  
Эти знания уже позволят составлять простые программы на JavaScript (см. теорему Бёма — Якопини).  
На этом уровне возможно и даже нужно остановиться на некоторое время для создания неразрушимого фундамента из знаний и навыков.  
На этом шаге нужно усвоить смысл Парадигмы Императивного Программирования.
